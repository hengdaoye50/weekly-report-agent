# 周报自动生成智能体 - 设计文档

## 一、项目概述

### 服务对象
需要每周撰写工作周报的学生或职场人士。

### 解决的问题
手动写周报存在以下痛点：
- 每周重复劳动，耗时15-30分钟
- 容易遗漏本周做过的事情
- 写出来的内容像流水账，缺乏结构
- 写完后还要手动发通知给相关人

### 输入
飞书文档中记录的本周工作内容（自由格式的文本记录）。

### 输出
- 结构化周报（按类别分组、专业润色、含下周计划和总结）
- 飞书文档（自动创建）
- 飞书消息通知（通过webhook）
- 命令行展示

## 二、处理步骤

```
Step 1: 收集 — 飞书API读取源文档 raw_content
Step 2: 分类 — LLM将工作内容按项目/类别自动分组
Step 3: 润色 — LLM对每组内容生成简洁专业的周报条目
Step 4: 组装 — LLM拼接完整周报（标题、分类内容、下周计划、总结）
Step 5: 输出 — 飞书文档 + webhook通知 + CLI打印
```

## 三、模型任务 vs 工具任务

| 环节 | 执行者 | 说明 |
|------|--------|------|
| 读取文档 | 飞书API | docx v1 raw_content 接口 |
| 内容分类 | 大模型 | 识别工作类别，分组归类 |
| 条目润色 | 大模型 | 将流水账转化为专业表述 |
| 周报组装 | 大模型 | 生成完整结构化周报 |
| 创建文档 | 飞书API | docx v1 创建文档接口 |
| 发送通知 | 飞书Webhook | 自定义机器人webhook |
| CLI展示 | Python | click + 格式化输出 |

## 四、异常处理

| 异常情况 | 处理方式 |
|----------|----------|
| 源文档为空 | 报错提示，要求检查文档内容 |
| 飞书API调用失败 | 抛出异常，显示错误码和信息 |
| LLM返回非法JSON | 重试一次，仍失败则将原始文本作为输出 |
| 飞书凭证未配置 | 启动时检查，提示填写.env |
| webhook地址为空 | 跳过通知，只输出文档和CLI |

## 五、技术栈

- **Python 3.12+**
- **lark-oapi**: 飞书官方Python SDK，自动管理token
- **openai**: LLM调用（兼容DeepSeek等第三方API）
- **click**: CLI框架
- **httpx**: HTTP客户端（webhook调用）
- **python-dotenv**: 环境变量管理

## 六、配置说明

在项目根目录创建 `.env` 文件（参考 `.env.example`）：

1. 在[飞书开放平台](https://open.feishu.cn/)创建应用，获取 App ID 和 App Secret
2. 给应用开通 `docx:document` 权限
3. 创建自定义机器人，获取 webhook 地址
4. 在[DeepSeek](https://platform.deepseek.com/)获取 API Key
5. 将以上信息填入 `.env`
